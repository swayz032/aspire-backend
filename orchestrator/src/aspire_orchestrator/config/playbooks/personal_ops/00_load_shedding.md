# Playbook: Load Shedding (Overwhelm → Execution)

## Purpose
When the owner is overloaded, reduce cognitive load without pretending to be therapy.

## Use when
- User says overwhelmed, stressed, behind, exhausted
- Task backlog is large or unclear

## Steps
1) **Name the load** (what’s causing the stress):
   - Unclear priorities, too many commitments, inbox pressure, cash fear, team issues

2) **Choose one loop to close today**
   - Pick the loop with the highest cost of delay.

3) **Reduce commitments** (one decisive cut)
   - Cancel/decline one low-value meeting or pause one project.

4) **Delegate**
   - Draft delegation message to team or vendor.

5) **Protect a recovery block**
   - Schedule a short personal block (walk, meal, reset) as an operational requirement.

## Output pattern (notes)
- The load (1–2 lines)
- Today’s single closure target
- One cut (meeting/project)
- Delegation draft
- Recovery block suggestion
