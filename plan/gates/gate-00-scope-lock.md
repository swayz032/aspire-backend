# GATE 00: Scope Lock

## Requirement

Aspire Founder Console, Founder Quarter only. No metaverse/spatial elements.

## Strategic Positioning

**What Aspire IS:**
- Execution Infrastructure (governed AI labor)
- Business Operations Platform (Founder Quarter focus)
- Accountability Substrate (receipts + approvals + staged autonomy)

**What Aspire IS NOT:**
- Communication tool (no Slack/Teams competitor)
- Metaverse platform (no spatial/Unity elements in v1)
- AI chatbot (no unstructured conversation without execution)

## Strategic Pivot v3.0

- **Unity desktop-only** (no mobile Unity, no WebGL, no VR/AR)
- **"Aspire City" internal legacy** (removed from public positioning)
- **Execution Infrastructure positioning** (foundational narrative)

## Verification Criteria

- [ ] No metaverse/spatial features in v1 roadmap
- [ ] Founder Quarter is ONLY market segment for v1
- [ ] Strategic positioning consistently uses "Execution Infrastructure" language
- [ ] Unity desktop NOT mentioned in external marketing
- [ ] Mobile app is React Native (Expo), not Unity

## What This Gate Prevents

- **Scope creep** - Adding unrelated features that dilute positioning
- **Market confusion** - Competing with wrong category (communication vs. execution)
- **Resource waste** - Building metaverse features before proving core value

## Failure Scenarios

❌ **Fails if:**
- v1 roadmap includes spatial/3D features
- Marketing positions Aspire as "AI chat" or "communication tool"
- Founder Quarter is not the exclusive v1 target

✅ **Passes if:**
- Roadmap focuses exclusively on business operations automation
- Positioning is clear: Execution Infrastructure for solopreneurs
- All features tie back to governed execution (not conversation)
